clear_align_node_iteration: bool¶

Whether to clear (_Context.clear()) the intermediate data structures after each iteration of the node alignment algorithm. Cursory results on a small sample show flow and alignments only slightly change when _Context.neigh_sim is cleared.

This is because the concept embeddings are used in network neighborhood calculations and a convergance algorithm (like CRFs) are implemented at this time.

Setting to False shows a speeds it up by ~75% on a small random sample.

embedding_populator: EmbeddingPopulator¶

Adds embeddings to frameset instances and used for debugging in this class.

hyp: HyperparamModel¶

The calculator’s hyperparameters.

Parameters:

  • similarity_dampen_exp (dict) – the exponent that dampens the cosine similarity for all nodes

  • concept_embedding_role_weights (dict) – the weights for concept node’s roles (i.e. arg0 with neighbor node)

  • concept_embedding_weights (dict) – the weights for concept node’s weighted average across tokens and role sets

  • neighbor_direction (str; one of: descendant, ancestor, all) – indicate how to find neighbors (from the point of the reverse/maxflow graph with the root on the bottom), which is one of descendant (only children and descendents), ancestor (only parents and ancestors), or all (all neighbors)

  • neighbor_embedding_weights (list) – weights used to scale each neighbor from the the current node and immediate neighbor to the furthest neighbor; if there is only one entry, the singleton value is multiplied by the respective nodes embeddings

  • neighbor_skew (dict) – neighborhood sidmoid skew settings with y/x translation, and compression (how much to compress or “squeeze” the function to provide a faster transition) with cosine similarity as the input

  • sentence_dampen (float) – the slope for the linear dampening of nodes under a sentence by sentence cosine similarity; the higher the value the lower the sentence similarity, which leads to lower concept and attribute node similarities, must be in the interval [0, 1]

include_tokens: bool¶

Whether to include tokens in the dataframe for debugging.
